🗺️ THE ROAD MAP

Your Coding Journey

From "What is an instruction?" to designing real systems — every great coder walked this exact path.

Step 1
Instructions & Sequences

Everything a computer does starts with one instruction at a time. A sequence is a list of steps in exact order — like a recipe or directions to the park. Computers follow sequences perfectly and never skip or guess.

Step-by-stepOrder mattersNo ambiguity
📋
❓
Step 2
Decisions & Conditions

Real programs must make choices. "If it is raining, take an umbrella — otherwise wear sunglasses." This YES/NO thinking is called a conditional and is the most powerful idea in all of computing.

IF / ELSETrue / FalseMultiple paths
Step 3
Loops & Repetition

Computers love doing things over and over without complaining. A loop says "repeat this until I say stop." Without loops, you'd write the same instruction 1,000 times — loops make you powerful.

REPEAT n timesWHILE conditionFOR EACH item
🔁
📦
Step 4
Variables & Memory

A variable is a labelled box in the computer's memory. You put a value in, give it a name, and use or update it later. Score = 0. Collect a coin → Score = Score + 1. That's a variable in action.

Store dataUpdate itRead it back
Step 5
Functions & Reuse

A function is a mini-program you name and reuse. Instead of writing "wash hands" steps every morning, noon and evening — write it once, call it WASH_HANDS, and call it three times. DRY: Don't Repeat Yourself.

Define onceCall anywhereDRY principle
🛠️
📋
Step 6
Lists & Collections

What if you need to store 100 names? Use a list — one box with numbered compartments. Lists let you group related things, search through them, sort them and process every item with a loop.

CollectionsIndex from 0Search & Sort
Step 7
Algorithms & Thinking

An algorithm is a smart plan to solve a problem efficiently. How do you find the biggest number? How do you sort 1,000 names A–Z? Algorithmic thinking solves any problem — in code or in life.

SearchSortEfficiencyOptimise
🧠
🐛
Step 8
Debugging & Problem Solving

Every program has bugs — mistakes that make it do the wrong thing. Debugging is detective work: find the clue, trace the logic, isolate the problem, fix it. The first real computer bug was an actual moth stuck inside a machine in 1947!

Find the errorTest & traceFix & verify

The Big Ideas

Master these concepts and you can build anything — in any language, on any machine, in any era.

📋
FOUNDATION
Sequences

A computer executes instructions one at a time, in exact order. There is no guessing, no skipping — each step must be crystal clear.

🍕 Making pizza — as an algorithm:
1. Preheat oven→2. Spread sauce
3. Add toppings→4. Bake 15 min
⚠️ Do step 4 before step 1? Disaster! Order always matters.
START
  MOVE forward 3 steps
  TURN right
  MOVE forward 2 steps
  PICK UP treasure
END
❓
CORE
Conditions (IF/ELSE)

A condition checks if something is TRUE or FALSE, then takes a different path for each answer. This lets programs react to the world.

🌧️ Weather decision tree:
Is it raining?
YES ✓→Take umbrella ☂️
NO ✗→Wear sunglasses 😎
IF score >= 90:
  SAY "Outstanding! 🏆"
ELSE IF score >= 60:
  SAY "Well done! 🌟"
ELSE:
  SAY "Keep going! 💪"
🔁
CORE
Loops

Loops tell the computer to repeat a block of instructions — a fixed number of times, or until a condition is met. They are the engine of automation.

🌅 Morning routine loop:
REPEAT 5 times:
Wake up → Eat → Go to school
END REPEAT
REPEAT 10 times:
  DRAW a star ⭐
  MOVE right 20px

WHILE energy > 0:
  PLAY game
  energy = energy - 1
📦
CORE
Variables

Variables are named containers for data. The value inside can change — that's the point! They let programs remember, update and use information.

🎮 Game score example:
score = 0→ collect coin →score = 1
→ collect coin →score = 2
The label stays ("score") but the VALUE inside changes!
name = "Priya" ← text
lives = 3 ← number
hasKey = TRUE ← yes/no

lives = lives - 1 ← update!
🛠️
INTERMEDIATE
Functions

A function is a named block of logic you define once and call many times. Functions take inputs (parameters) and can return outputs — like a machine.

🏭 A function is a machine:
Input: 5→[DOUBLE machine]→Output: 10
Same machine, different inputs → different outputs!
DEFINE greet(name):
  SAY "Hello, " + name + "!"

greet("Arjun") → "Hello, Arjun!"
greet("Riya") → "Hello, Riya!"
🧠
ADVANCED
Algorithms

An algorithm is a step-by-step plan to solve a problem efficiently. The same problem can have many algorithms — some are much faster and smarter than others.

🔍 Find the biggest number in a list:
1. Assume the first number is biggest
2. Check each other number in order
3. If it's bigger, remember it instead
4. After all checked → you have the winner!
Works for 10 numbers or 10 million!
biggest = list[0]
FOR EACH num IN list:
  IF num > biggest:
    biggest = num
SAY biggest
🔧
ADVANCED
Decomposition

Break a big, scary problem into smaller manageable pieces. This is how real engineers tackle enormous projects — divide and conquer, always.

🎮 Building a game — broken down:
"Build a game"→ too big!
Draw screen
Move player
Track score
Check win
Big problem → small functions:
setupGame()
drawScreen()
movePlayer()
checkScore()
checkWin()
🐛
ESSENTIAL
Debugging

Every coder writes bugs. Debugging is finding exactly WHY something is wrong. Read the error, trace the logic, isolate the problem, then fix it and verify.

🔍 Debug detective steps:
1. 🛑 Read the error message carefully
2. 🗺️ Find which line caused it
3. 🤔 Expected vs actual — what differs?
4. 🔬 Test one small piece at a time
5. ✅ Confirm the fix actually works
❌ BUGGY — spot the error:
REPEAT 5 times:
  SAY score
  score = score + 1
↑ score was never set to 0!
